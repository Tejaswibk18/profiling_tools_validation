System Validation & Profiling Framework Documentation
1. Project Overview
This project is designed to automate the execution, validation, profiling, and reporting of different infrastructure and platform-related modules across multiple environments such as Linux, Windows, ARM, Cloud, GPU Nodes, and GPU Clusters.
The framework supports:
Platform Profiling 
Workload Profiling 
Meta Collection 
Capacity Sizing 
Validation of Nested Keys 
Execution with and without sudo 
ZIP/UNZIP handling 
HTML Reporting 
Automated Test Case Validation 
The framework is fully argument-driven using Python argparse and supports scalable execution across multiple servers and environments.

8. Key Validation Feature
The framework supports validation of deeply nested keys inside JSON payloads.
Example JSON:
{
  "chassis": {
    "systems": {
      "bios": {
        "version": "1.0"
     
 
#suppose this is newly added
      }
    }
  }
}
Key Input Format:
[chassis.systems.bios.version]
Supported Features:
Single key validation 
Multiple key validation 
Nested dictionary traversal 
Null value validation 
Missing key validation 
Validation Output:
Key Present 
Key Missing 
Key Value is Null 
Example Messages:
[PASS] Key found: chassis.systems.bios.version
[FAIL] Key missing: chassis.systems.bios.version
[FAIL] Key exists but value is null
